# 小鸿SE智能家居中控系统

本项目基于星闪SLE协议和OpenHarmony LiteOS-M操作系统，实现了一套一主多从的智能家居中控系统。系统由P4屏幕作为UI和控制中心，WS63 Hub作为星闪网关，同时连接四个子设备模组，支持手动控制与自动联动两种模式。

| 设备 | 芯片 | 外设 | 广播名 | MAC尾缀 |

## 四、核心功能

### P4端UI
- **环境页**: 温湿度显示、人体感应状态、各模组在线状态、当前场景模式
- **控制页**: 风扇三档控制(关/低/中/高)、照明三档控制(关/低/中/高)
- **场景控制**: 居家/离家/睡眠三种场景手动切换 + 自动模式(有人→居家，无人→离家)
- **按钮彩色区分**: 低-蓝、中-黄、高-红、关-灰

### 通信协议
- **UART帧格式**: `A5A55A5A` + checksum + cmd + len + version + `78563412` + payload
- **扩展命令**: 0x0F12控制、0x0F13场景、0x0F14人体、0x0F15风扇、0x0F16照明
- **TLV控制协议**: seq + module + msg_type + tlv_type + value

### 联动逻辑
| 联动规则 | 触发条件 | 执行动作 |
|---|---|---|
| 温度→风扇 | 温度≥30℃ | 自动开启风扇，随温度升高增加档位 |
| 人体→场景 | 自动模式+有人 | 切换到居家场景 |
| 人体→场景 | 自动模式+无人30秒 | 切换到离家场景 |

## 七、通信协议扩展

本系统在原始代码基础上扩展了以下命令号：

命令	                             值	     方向	          说明
OHOS_UART_CMD_SENSOR_CONTROL	    0x0F12	P4 → 模组	  控制命令（风扇/照明档位）
OHOS_UART_CMD_SCENE_CONTROL	      0x0F13	P4 → 模组 	场景切换命令
OHOS_UART_CMD_SENSOR_HUMAN_REPORT	0x0F14	模组 → P4 	人体感应上报
OHOS_UART_CMD_SENSOR_FAN_REPORT	  0x0F15	模组 → P4 	风扇状态上报
OHOS_UART_CMD_SENSOR_LIGHT_REPORT	0x0F16	模组 → P4 	照明状态上报
TLV控制协议格式:

payload: seq(1) + module(1) + msg_type(1) + tlv_type(1) + value(1)
示例: 0x01 0x03 0x03 0x21 0x02  → 风扇设为中档
